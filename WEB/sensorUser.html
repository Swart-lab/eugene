USER information
The prediction can take into account specific user information (usefull to explore alternative gene structures). A simple language is required, described bellow, containing statements on signals and on the sequence itself :
Statements on signals Statements on sequence itself
type start stop acceptor donor type exon intron utr5 utr3 intergenic
strand f = forward - r = reverse strand f = forward - r = reverse
  frame Only used for exons - introns
The frame of an exon indicates the position of the codons wrt. to the start-end of the sequence (modulo 3, same as reported by BlastX).
The frame of an intron specifies how the codon is split by the splice site : between 2 codons, after 1 nucleotide or after 2. Basically, if you don't care of the intron "frame" just put the same information for all 3 frames.
position Position on the sequence [start..end] Scope of the information
value Cost associated with the signal.
It can range from 0.0 to 1.0. A value of 0.0 means that no site of this type exists at this position. The cost will override any existing cost. 		value Score that can take any floating point value plus the extra values "infinity" and "-infinity". This cost will be added to the existing cost.
A positive cost helps predicting something consistent with the information.
A negative one prevents from predicting something consistent with the information. Thus a cost "-infinity" effectively forbids the prediction.
